### 📝 1. Writer, Designer, Developer 간의 공식 협업 단계 (4단계 피드백 루프)

다음은 모든 결과물이 일관성을 유지하며 최종 목표를 향해 나아가도록 하는 4단계 피드백 루프입니다.

**🚀 실행 계획: Trust Score Landing Kit 구현을 위한 4단계 피드백 루프**

1.  **[Phase 1: 가치 제안 & 검증] (Writer & 현빈 중심)**
    *   **목표:** 핵심 가치 제안(Value Proposition) 및 카피의 심리적 타당성 확정.
    *   **입력:** Writer의 카피, 현빈의 수익화 전략.
    *   **출력:** 최종 H1/Sub-Headline 및 CTA 결정.

2.  **[Phase 2: 시각화 & 레이아웃 설계] (Designer 중심)**
    *   **목표:** 확정된 가치 제안에 기반한 디자인 시스템 적용 Mockup 제작.
    *   **입력:** Phase 1의 최종 카피 및 핵심 메시지.
    *   **출력:** Trust Score 강조 Hero 섹션 Mockup (디자인 지침 포함).

3.  **[Phase 3: 기능 & 데이터 모델 구현] (Developer 중심)**
    *   **목표:** Phase 2 디자인과 Phase 1 가치 제안을 반영한 핵심 기능 로직 및 데이터 모델 구현.
    *   **입력:** Phase 2의 레이아웃 스펙, Trust Score 연동 로직 요구사항.
    *   **출력:** Trust Score 연동 로직이 포함된 DB 스키마 및 API 설계 초안.

4.  **[Phase 4: 통합 검증 & 최종 확정] (모든 에이전트 참여)**
    *   **목표:** 디자인, 카피, 기능의 일관성(Alignment)을 최종적으로 검증하고 실행 계획을 공식화.
    *   **입력:** Phase 3의 구현 결과물.
    *   **출력:** 통합 QA 보고서 및 다음 개발/디자인 스프린트 계획.

### 📋 2. 통합 보고서 템플릿 (배포 준비 완료)

이 템플릿은 각 에이전트가 산출물을 제출할 때 사용하며, **Phase 4**에서 최종 검증에 사용됩니다.

#### **[통합 실행 계획 보고서 템플릿]**

**프로젝트:** make waves - Trust Score Landing Kit 구현
**날짜:** YYYY-MM-DD
**담당자:** (Writer / Designer / Developer)
**목표 단계:** [Phase 1, 2, 3, 또는 4 중 선택]

**🎯 1. 핵심 요약 (Executive Summary)**
*   이번 작업의 가장 중요한 결정 사항은 무엇인가? (단 한 문장으로 요약)
*   이 결과물이 전체 목표에 기여하는 바는 무엇인가?

**✍️ 2. 상세 산출물 검토 (Detailed Deliverables Review)**
*   **[Writer/마케팅]**: 최종 확정된 카피 및 가치 제안 명시.
    *   (예: H1, Sub-Headline, CTA)
*   **[Designer]**: 적용된 디자인 시스템 및 레이아웃의 주요 결정 사항.
    *   (예: 사용된 컬러 코드, 핵심 시각화 요소 위치)
*   **[Developer]**: 구현한 데이터 모델 또는 API 구조의 최종 버전.
    *   (예: Trust Score 테이블 정의, 핵심 API 엔드포인트)

**✅ 3. 통합 검증 항목 (Alignment Check)**
*   **일관성 점검:** 카피/디자인/기능이 '심리적 안정감'이라는 핵심 가치를 일관되게 전달하는가? (Yes/No 및 근거 명시)
*   **기술 구현 가능성:** 제시된 디자인과 데이터 모델을 기반으로 개발팀의 실행 가능성은 충분한가? (Yes/No 및 위험 요소 명시)

**➡️ 4. 다음 액션 플랜 (Next Action Plan)**
*   **다음 단계:** [Phase 1 → Phase 2] 또는 [Phase 3 → Phase 4] 중 선택하여 명확하게 제시.
*   **필요한 지원:** (예: "디자인팀은 Trust Score 시각화에 대한 추가 데이터 요청", "개발팀은 API 사양을 상세화 요청")
