## 🎯 수익화 모델 및 MVP 우선순위 재검토 보고

### 1. 수익화 모델 타당성 검토 (Trust Score 기반 VBP)

| 항목 | 기존 전략 | 재검토 결과 | 권고 사항 |
| :--- | :--- | :--- | :--- |
| **가격 전략** | Value-Based Pricing ($49.99 ~ $69.99) | **타당성 높음.** 외국인 대상의 '불안감 해소'라는 심리적 프리미엄을 가격에 반영할 수 있음. | **유지.** 초기에는 3단계(Basic, Pro, Enterprise)로 세분화하여 전환율을 테스트해야 함. |
| **핵심 가치** | 신뢰도 점수 (Trust Score) | **매우 강력함.** 고객의 불안감을 측정 가능한 지표로 전환하는 것은 명확한 Pain Point 해결책임. | Trust Score가 단순한 점수가 아니라, '신뢰 기반의 행동 유도'라는 기능으로 포장되어야 함. |
| **수익화 검증** | Landing Kit MVP 구현 | **필수.** 실제 결제 흐름이 작동해야만 가격 책정의 타당성을 확인할 수 있음. | MVP는 복잡한 기능보다 **"신뢰 점수를 제시하고 구독하도록 유도하는 단일 흐름(Single Flow)"**에 집중해야 함. |

### 2. MVP 기능 우선순위 결정 (Actionable Prioritization)

개발팀과 비즈니스 목표를 고려하여, 개발 환경 안정화가 완료되는 시점까지 **최소한의 실행 가능한 MVP**를 정의합니다.

| 우선순위 | 기능 모듈 | 목표 (MVP 달성 기준) | 전략적 이유 |
| :--- | :--- | :--- | :--- |
| **P1 (Must Have)** | **Trust Score 연동 및 표시** | 사용자가 자신의 현재 '신뢰도 점수'를 명확하게 인지하고 시각화하는 기능. | 핵심 가치(Value Proposition)의 존재 이유. 이 기능 없이는 수익화가 불가능함. |
| **P2 (Should Have)** | **단순 구독 및 결제 흐름** | Trust Score 기반으로 $49.99/월 또는 $69.99/월 옵션을 선택하고 결제할 수 있는 최소한의 프로세스. | 수익화 모델 검증의 핵심. 기능이 아닌 '거래'가 성공해야 비즈니스 모델이 증명됨. |
| **P3 (Could Have)** | **AI 기반 맞춤형 로드맵 생성** | Pro Tier에 포함될 AI Q&A 기록 및 맞춤형 로드맵 생성 기능. | 추가적인 프리미엄 가치(심리적 안정감)를 제공하지만, MVP에서는 제외하고 P2 성공 후 확장할 것. |

개발팀의 환경 설정 안정화 ETA를 기다리는 동안, 다음 두 가지 작업을 **동시에** 실행하여 시간을 확보합니다.

    *   **목표:** 환경 설정 안정화가 완료되는 즉시 MVP 코드를 배포할 수 있도록 기술적 구현을 앞당긴다.
