# 🚀 Pro Tier API 계약 조건 및 협의 준비안 (Trust Score 기반)

## 1. Trust Score 및 가격 책정 근거 정의

Pro Tier 구독 모델은 단순 기능 제공을 넘어 **'신뢰(Trust)'를 기반으로 한 맞춤형 가치와 리스크 감소**에 대한 프리미엄을 받는 구조로 설정합니다.

### A. 수익화 근거 (Pricing Rationale)
*   **가치 측정:** Trust Score는 사용자의 실제 경험과 데이터의 질을 객관적으로 반영하는 핵심 지표입니다. 이 점수를 기반으로 제공되는 맞춤형 결과물(로드맵, Q&A 기록)에 대한 **'확신성 프리미엄(Certainty Premium)'**을 가격 책정 근거로 삼습니다.
*   **ROI 연결:** 높은 Trust Score는 사용자의 성공 확률을 높이고 이탈률을 낮추므로, API 사용료는 단순 트랜잭션 비용이 아닌 **사용자 성공 지원 서비스 비용**으로 포지셔닝합니다.

## 2. Pro Tier API 기능 요구사항 정의 (Feature Set)

Pro Tier에 포함되어야 할 핵심 기능과 데이터 범위를 명확히 정의하여 코다리와의 협의를 준비합니다.

### A. 핵심 기능 요구사항 (Features)
1.  **Trust Score 기반 로드맵 생성:** 사용자의 현재 상태(입력된 데이터)와 Trust Score를 입력값으로 받아, **최적화된 맞춤형 다음 단계 액션 플랜**을 생성하는 API 엔드포인트 제공.
2.  **AI Q&A 기록 저장 및 분석:** 사용자가 AI와 주고받은 모든 상호작용(Q&A)을 안전하게 저장하고, 이 기록을 바탕으로 **사용자의 학습 패턴과 신뢰 변화 추이**를 분석하여 피드백을 제공하는 기능.
3.  **신뢰도 기반 위험 경고 (Trust-based Alert):** Trust Score가 특정 임계값 이하로 하락했을 경우, 시스템이 잠재적 위험 요소를 식별하고 사용자에게 **선제적인 조치 권고**를 API를 통해 제공.

### B. 데이터 범위 및 접근성 (Data Scope)
| 데이터 항목 | 범위 및 포함 내용 | 접근 권한 | 보안 중요도 |
| :--- | :--- | :--- | :--- |
| **Trust Score** | 산출된 최종 점수, 점수 변화 추이, 주요 영향 요인 분석 (숨겨진 가중치 포함) | Pro Tier 사용자에게만 제공 (Read-Only) | 최상 |
| **사용자 입력 데이터** | 로드맵 생성에 사용된 초기 입력값 및 중간 단계 데이터 | Pro Tier 사용자 본인만 접근 가능 | 높음 |
| **Q&A 기록** | 모든 대화 내용, 시스템의 추천 답변, 사용자의 수정한 기록 | Pro Tier 사용자에게만 접근 가능 | 중간 |
| **시스템 로그** | API 호출 기록 및 내부 처리 과정 (보안상 제외) | 관리자(혹은 자동화된 분석 엔진)에게만 접근 | 최상 |
