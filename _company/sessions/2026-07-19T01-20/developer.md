기존에 확정된 데이터 모델(`models.py`)과 비즈니스 요구사항을 기반으로, 기능의 복잡도와 사용자 경험(UX) 기여도를 고려하여 개발 우선순위를 설정합니다.

#### 2. 상세 기술 실행 계획 (API 엔드포인트 및 데이터 흐름)

**전제:** 사용자는 `models.py`에 정의된 PostgreSQL 스키마(예: `roadmap_plans`, `qa_sessions` 등)를 기반으로 API가 동작한다고 가정합니다.

##### Phase 1: MVP 핵심 기능 구현 (P1 & P2) - 데이터 중심
**목표:** Trust Score의 정확한 계산 및 저장, 그리고 이를 조회하는 기본 엔드포인트 확보.

*   **데이터 모델 검증:** `models.py`에 정의된 테이블 구조와 관계가 API 로직과 완벽히 일치하는지 최종 확인합니다. (P1 실행 전 필수)
*   **API 1: Trust Score 계산 및 저장 (`POST /api/v1/scores`)**
    *   **입력:** 사용자 입력 데이터 (예: 신뢰도 점수 요소들).
    *   **처리:** 비즈니스 로직(`models.py` 기반)을 실행하여 최종 `TrustScore`를 계산하고 DB에 저장합니다.
    *   **출력:** 저장된 Trust Score 및 관련 메타데이터 반환.
*   **API 2: 사용자 점수 조회 (`GET /api/v1/user/score/{user_id}`)**
    *   **입력:** `user_id`.
    *   **처리:** DB에서 해당 사용자의 최신 Trust Score 및 관련 기록을 조회합니다.
    *   **출력:** 사용자에게 표시할 점수 데이터 반환.

##### Phase 2: Pro Tier 기능 구현 (P3 & P4) - 로직 중심
**목표:** 맞춤형 로드맵 생성 및 AI Q&A 기록 기능을 위한 복잡한 비즈니스 로직 API 구축.

*   **API 3: 맞춤형 로드맵 생성 (`POST /api/v1/roadmap`)**
    *   **입력:** 사용자의 목표, 현재 상태 데이터.
    *   **처리:** **(핵심)** 정의된 로드맵 생성 알고리즘을 실행하고, AI 모델과의 연동 지점(Prompt Engineering)을 통해 로드맵 초안을 생성합니다. (P1의 결과 활용)
    *   **출력:** 생성된 로드맵 구조 데이터 반환.
*   **API 4: AI Q&A 기록 (`POST /api/v1/qa_session`)**
    *   **입력:** 사용자 질문, 답변(또는 시스템 응답).
    *   **처리:** 질문과 답변을 DB(`qa_sessions` 테이블)에 저장하고, 필요하다면 AI 모델을 통해 Q&A 기록을 구조화하여 저장합니다. (P4 실행)
    *   **출력:** 저장된 세션 기록 반환.
