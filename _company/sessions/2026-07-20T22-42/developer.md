CEO의 지시에 따라 Designer가 확정한 시각 시스템(Deep Ocean Blue, Aqua Teal)과 USP를 기반으로 백엔드 API 및 UI/UX 일관성을 점검하는 통합 QA 프로세스를 재개하기 위한 테스트 계획을 준비하겠습니다.

이 QA는 **데이터 로직(Backend)**과 **시각적 구현(Frontend Spec)** 간의 일관성 확보에 중점을 둡니다.

### 📝 통합 QA 테스트 계획 (Trust Score Gauge)

**목표:** Trust Score Gauge 솔루션의 백엔드 API와 프론트엔드 디자인 스펙 간의 End-to-End 일관성을 검증한다.
**기준:** 확정된 시각 시스템 (`#0A1F38`, `#2DE3C2`) 및 핵심 USP('완벽하게 설계된 성공 경험').

#### 1. 데이터 모델 및 API 무결성 검증 (Backend Focus)

이 단계는 `models.py`와 FastAPI 엔드포인트가 디자인 스펙에 맞는 데이터를 정확히 반환하는지 확인합니다.

*   **테스트 항목 A: Trust Score 데이터 일관성**
    *   **검증 내용:** 백엔드에서 계산된 `Trust Score` 값이 프론트엔드에서 요구하는 WTP 기반의 프리미엄 가치(Pricing Tier)와 정확히 매핑되는지 확인한다.
    *   **실행 방안:** `models.py` 내의 핵심 로직(`roadmap_plans`, `qa_sessions` 테이블 관련 필드)을 검토하고, 각 티어($19.99, $49.99)에 따른 Score 계산 공식이 디자인에서 제시된 WTP 가설과 일치하는지 확인한다.
*   **테스트 항목 B: API 응답 형식 및 컨텍스트 전달**
    *   **검증 내용:** 모든 핵심 엔드포인트(`GET /scores`, `GET /roadmap`)가 프론트엔드가 기대하는 JSON 구조와 필드를 완벽하게 반환하는지 검증한다. (데이터 누락 또는 오분류 방지)
    *   **실행 방안:** FastAPI 엔드포인트별로 Mock 데이터를 사용하여 응답 형식을 사전 검증하고, 실제 데이터 흐름에서 예외 상황(Empty State 등) 처리 시의 응답도 확인한다.

#### 2. UI/UX 일관성 검증 (Frontend Focus)

이 단계는 디자인 스펙이 실제로 구현될 때 발생하는 잠재적 불일치를 미리 식별합니다.

*   **테스트 항목 C: 색상 및 타이포그래피 적용 검증**
    *   **검증 내용:** Hero 섹션의 Primary Background Color (`#0A1F38`, Deep Ocean Blue)와 Accent Color (`#2DE3C2`, Aqua Teal)가 전체 랜딩 페이지에 일관되게 적용되었는지 시각적으로 확인한다.
    *   **실행 방안:** 컴포넌트 레벨에서 색상 변수(CSS Variables 또는 Theme Context)를 사용하여 모든 핵심 섹션의 배경과 강조색이 지정된 팔레트를 따르는지 검증한다.
*   **테스트 항목 D: 정보 계층 구조 (Hierarchy) 검증**
    *   **검증 내용:** H1, H2, CTA 버튼 등의 타이포그래피 크기 및 폰트 패밀리(`Poppins`, `Inter`)가 모바일 환경에서 명확한 시각적 위계(Visual Hierarchy)를 형성하는지 확인한다.
    *   **실행 방안:** 반응형 디자인 테스트(Mobile-First)를 실행하여, 작은 화면에서도 메시지의 중요도 순서가 의도대로 전달되는지 검증한다.

#### 3. 통합 E2E 흐름 검증 (End-to-End Flow)

이 단계는 사용자가 실제로 경험하는 전체 여정을 테스트합니다.

*   **테스트 항목 E: 성공 경로 확보 플로우 검증**
    *   **검증 내용:** 사용자가 Trust Score를 확인하고, 그 결과에 따라 다음 행동(예: Pro Tier 업그레이드 CTA 클릭)을 했을 때, 백엔드 데이터가 즉각적이고 논리적으로 반영되는지 확인한다.
    *   **실행 방안:** 최소 2가지 시나리오(성공적인 경로 및 실패/불일치 경로)에 대해 API 호출부터 UI 업데이트까지의 지연 시간과 데이터 정확성을 측정하고 기록한다.
